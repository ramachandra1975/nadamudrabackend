Raga Universe Scale

No fixed maximum: in theory, ragas can be created in very large numbers through different swara selections, arohanam/avarohanam patterns, vakra movement, gamakas, and note emphasis.

Carnatic: about 30,000+ possible ragas are often estimated through Melakarta–Janya combinations.

Hindustani: roughly 3,000 active and theoretical ragas are commonly cited, depending on classification.

In practice, only a smaller number become established and musically meaningful.

User Guide
How to use Nāda Mudrā V9 from scratch
Nāda Mudrā V9 is a complete interactive platform for exploring rāgas, swaras, tālas, mudrās, kritis, mantras, bījāksharas, music theory, history, and rāga creation in one place. The app is designed for learning, searching, comparison, reference, and creative exploration. Users can browse information, search structured content, interact with musical models, and generate new rāga ideas through guided tools. The app is organized through a top tab-based navigation system. Each tab opens a different section such as Rāga Library, Rāga Finder, Swara Engine, Mudrās, Kriti Library, Cross-Reference, Mind Science, Tāla, Time Wheel, Melakarta, Thaat, Mantra, Bījākshara, History, and Creator. Global controls at the top allow users to change the language/script display, switch the theme, print the current view, and adjust notation visibility.
1. General Navigation
Begin by clicking any tab in the tab bar to open that section. Only one pane is shown at a time, making it easier to focus on a specific part of the app. The language selector changes the notation or display labels, while the theme button changes the visual appearance. The print button can be used to print or save the current pane as PDF. Genre filters and notation toggles at the top help refine how information is viewed. These controls apply across relevant sections and allow users to personalize the display according to learning preference.
2. Rāga Library and Search
Use the Rāga Library to browse rāgas across Carnatic, Hindustani, Folk, and Fusion categories. Enter text into the search field to find rāgas by name, rasa, time, thaat, melakarta, or keyword. Use filters such as rasa and time to narrow the results. Use the separate Rāga Search pane for focused lookup. This section allows quick search and filtering by system, rasa, and time. Input in these sections is given through search boxes, dropdown filters, and clickable result cards.
3. Rāga Finder
The Rāga Finder helps identify rāgas from selected swaras. Users can click notes in the swara grid or use the piano keyboard to select and hear note positions. The Sa drone can be started for tonal reference. As notes are selected, the arohanam and avarohanam displays update. After selection, click Identify Rāga to see possible matches. Use Play Scale to hear the selected note pattern, Clear to reset, or Random to explore a surprise rāga. Result panels display rāga name, parent scale, rasa, season, time, gamakas, and confidence. Input here is done through clickable note grids, keyboard interaction, and action buttons.
4. Swara Engine, Tāla, and Theory
The Swara Engine provides reference information for swaras, shrutis, terminology, and notation across languages. The Tāla pane visualizes beat cycles and rhythmic patterns. The Mind Science pane explains theoretical, therapeutic, and scientific ideas connected to rāga and sound. These sections are mainly reference-driven. Users interact through reading, clicking visual elements, and observing structured panels, tables, or rhythmic displays.
5. Mudrās, Kritis, Mantras, and Bījāksharas
The app also includes cultural and spiritual knowledge sections. Hasta Mudrās can be explored by type and dance form. The Kriti Library allows searching compositions by name, rāga, composer, or genre. The Mantra and Bījākshara sections support search across sacred sounds, scriptural sources, deity associations, purposes, meanings, and related explanations. These sections accept input through search boxes, dropdown filters, category buttons, and clickable cards. Users can search, filter, and open entries to read detailed interpretive and reference material.
6. Compare, Time Wheel, and Constellation
The Compare pane lets users select two rāgas side by side using dropdown menus. The Time Wheel allows users to click time segments to see associated rāgas. The Constellation pane visually maps rāga relationships by rasa, time, parent system, or proximity. These sections are meant for analysis and visual understanding. Input is provided through dropdowns, clicks, and hover interaction.
7. Rāga Creator
The Rāga Creator is one of the app’s most advanced features. It allows users to create new rāga concepts by selecting arohanam and avarohanam swaras, choosing vādi and samvādi, applying gamakas, and setting the intended rasa and genre. Users may enter their own rāga name or leave the name field blank for an automatically suggested one. After entering the data, click Generate Rāga to produce a result. The output includes rāga type, rasa, genre, dominant notes, gamakas, ascending and descending structures, and interpretive analysis. Users can then play the scale, save the rāga, export it as JSON or text, or print it as PDF. This section demonstrates how rāgas can be created through note selection, movement structure, expressive identity, and musical intention.
8. How Data Is Entered in the App
Across the app, user input is given in several ways: text fields for search and naming, dropdown menus for filters and options, clickable swara grids for musical selection, buttons for actions such as identify, play, compare, save, and generate, and interactive cards or charts for deeper exploration. This means the app does not rely on long manual typing. Most of the interaction happens through structured visual controls, making the experience easier for both learners and advanced users.
9. End-to-End Use
A typical user journey may begin in the Rāga Library, continue into Rāga Finder for note-based exploration, move into Swara Engine or Mind Science for theory, then connect to Kritis, Mudrās, or Mantras for broader context. Advanced users can compare rāgas, study time associations, inspect parent systems, and finally create new rāga ideas in the Creator pane. In this way, Nāda Mudrā V9 functions as an end-to-end learning, exploration, reference, and creative platform that connects music, knowledge, structure, sound, and spiritual-cultural context in one integrated system.

User Guide
Complete step-by-step instructions for using Nāda Mudrā V9 from scratch
Introduction
Nāda Mudrā V9 is a complete interactive platform designed to help users explore, study, compare, and create knowledge related to rāgas, swaras, tālas, mudrās, kritis, mantras, bījāksharas, music theory, and history. The app is built as a unified knowledge and discovery system where musical structure, cultural context, spiritual content, and creative tools are all brought together in one interface. This application is useful for students, learners, musicians, dancers, teachers, researchers, devotional users, and anyone interested in Indian music and sound traditions. Some sections are designed mainly for reading and reference, while others are interactive and allow users to enter data, search, filter, compare, listen, or generate new outputs.
Step 1: Opening the App and Understanding the Layout
When the app opens, the top area contains the global controls. These remain important throughout the experience because they affect how the app is viewed and used. The first control is the language and script selector, which changes how musical terms or note names may be displayed. This allows users to view content in different Indian scripts or in Western notation. Next to it is the theme button, which changes the appearance of the app between visual modes such as dark and light display. The print button allows the currently open section to be printed or saved as a PDF. Beneath these controls are the genre filters and notation display toggles, which help users adjust which kind of content they want to focus on and how notation is shown. Below the top controls is the main tab bar. This is the primary way to move through the software. Each tab opens one section or pane of the app. Only one pane is shown at a time, so users can focus clearly on one topic without visual clutter.
Step 2: Navigating Through the Main Sections
The app is divided into multiple functional sections. These include Rāga Library, Rāga Finder, Swara Engine, Hasta Mudrās, Kriti Library, Cross-Reference, Mind Science, Tāla, Time Wheel, Melakarta, Thaat, Constellation, Compare, Rāga Search, Bījākshara Search, Mantra Search, History, and Rāga Creator. Each of these sections is designed for a different type of use. Some are for searching and filtering. Some are for reading and understanding theory. Others are for visual exploration, comparison, or interactive note selection. The Creator section goes even further by allowing users to construct new rāga ideas from scratch.
Step 3: Using the Rāga Library
The Rāga Library is the broad browsing area of the app. It helps users explore available rāgas across systems such as Carnatic, Hindustani, Folk, and related categories. At the top of the section, users can switch between system groupings such as All Systems, Carnatic, Hindustani, and Folk / Regional. Users can type into the search bar to look for a rāga by name, rasa, time, melakarta, thaat, or keyword. Additional filters such as rasa and time can be selected from dropdown menus. The results appear as cards, which visually summarize the rāga and its classification. This section is ideal for beginning exploration because it gives an overview of the content available in the app. Input methods used here: text search, dropdown filters, category buttons, and clickable result cards.
Step 4: Using the Rāga Finder
The Rāga Finder allows users to identify rāgas using swara selection. This section is interactive and useful for learners who want to understand how note combinations relate to rāga identity. To begin, the user clicks swaras in the visible swara grid. There is also a piano keyboard interface that can be used to hear note positions and support note selection. The user may also start the Sa drone to maintain a tonal reference while selecting or listening. As swaras are selected, the ascending and descending note displays begin to reflect the active selection. Once ready, the user clicks Identify Rāga to generate likely matches. The result area shows rāga name, alternate name, system, parent classification, time, season, rasa, gamakas, and confidence score. The user may then choose to play the scale, clear the selection, choose a random rāga, mark the result as a favourite, share it, or add it for comparison. Input methods used here: clickable swara grid, keyboard clicks, drone control, action buttons, and quick filters.
Step 5: Using the Swara Engine
The Swara Engine is a structured reference module for understanding note systems. It explains swaras across multiple languages and notation systems, helping the user understand note identity, relationship, terminology, and shruti placement. This section is not mainly for heavy input. Instead, it is designed for careful reading and browsing. Users can review swara cards to understand Carnatic naming, Western note equivalence, Solfège parallels, language forms, and pitch relationships. This is especially helpful when using the Finder or Creator sections because it builds note literacy. Input methods used here: primarily browsing, reading, and card inspection.
Step 6: Using the Hasta Mudrā Section
The Hasta Mudrā section focuses on classical hand gestures. Users can browse by major mudrā type such as Asamyutha, Samyutha, Therapeutic, or All Mudras. There are also dance-form tabs that let the user view mudrās within traditions such as Bharatanatyam, Kathak, Odissi, Kuchipudi, Manipuri, and Mohiniyattam. Mudrā cards present names, categories, descriptions, uses, and related interpretive meaning. This section is helpful for users interested in dance vocabulary, expressive symbolism, and embodied cultural meaning. Input methods used here: filter buttons, dance-form tabs, and clickable browsing.
Step 7: Using the Kriti Library
The Kriti Library helps users search and study compositions. Users can type composition names or keywords into the search field and narrow results using filters such as rāga, composer, and genre. Each composition card may include title, rāga, tāla, language, opening text, translation, composer, and descriptive notes. This section connects theoretical rāga study with actual compositional practice. Input methods used here: search field, dropdown filters, and clickable cards or action buttons.
Step 8: Using Cross-Reference
The Cross-Reference section is designed to show relationships between different entities in the app. A user begins by selecting a swara. Once selected, the app displays related rāgas, compositions, or associated concepts connected to that note. This section helps users move beyond isolated facts and understand musical relationships. It is especially useful for seeing how one element participates in a larger system. Input methods used here: clickable swara pills and interactive result selections.
Step 9: Using Mind Science
The Mind Science section provides interpretive and theoretical content about the effects of sound, rāga, psychoacoustics, neural response, rhythm, and therapeutic use. This pane contains concept cards and structured tables intended for reading and reflection. Users can study how rāgas are linked to emotional states, circadian patterns, neurological interpretation, and therapeutic associations. This section expands the app from musical classification into broader scientific and experiential understanding. Input methods used here: mainly reading, scrolling, and content review.
Step 10: Using Tāla, Time Wheel, Melakarta, and Thaat
These sections help users understand structural frameworks. In the Tāla pane, users explore rhythmic cycles and beat visualization. In the Time Wheel, users click time segments to see rāgas associated with different times of day. In the Melakarta pane, users explore the 72 parent rāga system of Carnatic music. In the Thaat pane, users study the parent scale system used in Hindustani music. Together, these sections build structural understanding. They explain how rāgas can be classified by note set, family, rhythm, or time tradition. Input methods used here: clicks on charts, grids, time segments, and visual controls.
Step 11: Using Constellation and Compare
The Constellation section visually maps relationships among rāgas. Users can choose whether to view those relationships by rasa, parent system, time, or proximity. Nodes can be hovered over or clicked to inspect related content. In the Compare section, users select two rāgas from dropdown menus and study them side by side. This is useful for understanding differences in scale, mood, time, or musical identity. Input methods used here: dropdown menus, view-selection buttons, node hover, and click interaction.
Step 12: Using Bījākshara and Mantra Search
The app also includes sections for sacred sound and textual reference. In the Bījākshara and Mātrikā Nyāsa section, users can search by syllable, deity, chakra, source, or purpose. In the Mantra and Shloka section, users can search sacred texts, nāmas, seed syllables, and related references. Detailed result panels display meaning, scriptural context, compositional relevance, spiritual significance, neurological interpretation, scientific view, and practical use. These sections expand the app beyond pure music classification into sacred sound knowledge and applied reference.
Step 13: Using the History Section
The History pane provides cultural and historical background for rāga systems. Users can read how ideas evolved from older traditions into later theoretical and practical forms. This section gives chronological depth to the app. It is primarily intended for reading and reference rather than active input. Input methods used here: reading and scrolling.
Step 14: Using the Rāga Creator from Scratch
The Rāga Creator allows users to build a new rāga concept step by step. First, the user selects notes for the arohanam or ascending movement. Next, the user selects notes for the avarohanam or descending movement. Then the user chooses the vādi and samvādi notes to define tonal emphasis. After that, the user may select one or more gamakas to shape musical movement and style. The user can then enter a rāga name or leave the field blank so the app can suggest one. Additional fields allow the user to define the intended rasa and genre. Once these inputs are ready, the user clicks Generate Rāga. The output shows the generated rāga’s type, rasa, genre, note structure, dominant notes, gamakas, and analytical description. The user can then play the scale, save the rāga, export it as JSON or text, or print it as a PDF. This section is important because it shows how rāgas can be created through the combination of note selection, ascending and descending structure, note emphasis, movement style, and intended emotional or aesthetic identity. It teaches that rāga creation is not only about scale choice, but also about musical personality and expressive logic. Input methods used here: swara selection grids, button toggles, name input field, dropdown menus, and generate/save/export controls.
Step 15: Understanding All Input Methods in the App
The app accepts user input in several structured ways. Text fields are used for searching and naming. Dropdown menus are used for filtering and selecting categories such as rasa, time, system, purpose, deity, source, or genre. Buttons are used for actions like identify, play, clear, randomize, compare, save, share, print, and generate. Clickable grids are used for swara selection and note construction. Cards and list items can be clicked to open deeper information. Visual interactive elements such as charts, wheels, and network maps allow users to explore relationships in a more intuitive way. This design ensures that the user does not need to manually type everything. Much of the experience is guided through structured interactive controls.
Step 16: Recommended End-to-End Way to Use the App
A beginner may start with the Rāga Library to become familiar with available material. From there, the user may move to the Rāga Finder to understand how note combinations identify rāgas. The Swara Engine and Mind Science sections can then be used to deepen theoretical understanding. After that, the user can connect musical knowledge with compositions in the Kriti Library, symbolic expression in Mudrās, and sacred sound reference in the Mantra and Bījākshara sections. Advanced users can compare rāgas, study time associations, review parent systems, and finally use the Rāga Creator to generate new concepts. In this way, the application supports a full journey from basic lookup to deep study, cross-reference, visual understanding, and creative experimentation.
Conclusion
Nāda Mudrā V9 is more than a reference library. It is a complete end-to-end exploration environment that brings together music, knowledge, cultural meaning, sound systems, rhythm, creativity, and sacred textual reference. Whether the user wants to search, learn, analyze, compare, or create, the app provides structured tools and guided interfaces to support the full experience from scratch.
